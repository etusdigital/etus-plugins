# Spike/Research Mode

Document a research investigation, feasibility study, or brainstorm. Generates a single spike document.

## Usage
```
/spike              -> Start research documentation
/spike [question]   -> Start with research question
```

## Workflow

1. Detect or confirm Spike mode
2. Run `/ideate` first in question-framing mode
3. Clarify the research question — what exactly needs to be answered?
4. Select methodology:
   - BMAD CIS brainstorm techniques available (all 8 techniques)
   - Comparative analysis (evaluate options with pros/cons)
   - Feasibility study (technical viability assessment)
   - Literature review (survey existing solutions)
5. Generate single spike-{slug}.md with:
   - **Research question** — The specific question to be answered
   - **Methodology** — How the investigation was conducted
   - **Findings** — What was discovered, with evidence
   - **Options evaluated** — Each option with pros/cons and tradeoffs
   - **Recommendation** — The suggested path forward with rationale
   - **Decision** — The final decision (if made during the spike)
   - **Next steps** — What work item to create next (feature, product, or bug)
6. Save to docs/ets/projects/{project-slug}/spikes/

## Output

```
docs/ets/projects/{project-slug}/spikes/
└── spike-{slug}.md    # Single document with all sections above
```

## Dependencies

- `opportunity-pack.md` — BLOCKS (generated by `/ideate`, mandatory)
- `coverage-matrix.yaml` — BLOCKS (generated by `/ideate`, mandatory)
- No upstream documents required.

## Validation

- Research question stated clearly
- Findings documented with evidence
- Recommendation present with rationale
- No quality gates.

## BMAD CIS Techniques

All 8 brainstorm techniques from the BMAD methodology are available:
- Use them to explore the problem space
- Multiple techniques can be combined in a single spike
- Findings from techniques feed directly into the Options and Recommendation sections

## Output Feeds Into

A spike can recommend creating:
- A **Feature** work item (`/feature`) if the spike validates a feature idea
- A **Product** work item (`/start-project`) if the spike reveals a larger opportunity
- A **Bug** work item (`/bugfix`) if the spike uncovers an existing defect
- Another **Spike** if the investigation raises new questions
